任务圈 TaskLoop / 共识集市 Consensus Market 产品需求文档（PRD）V1.0

1. 产品概述
   1.1 产品定位
   基于地理位置与社交关系的去中心化任务协作与轻量级价值交换平台。支持个人信用、小团体 consensus、全球通用货币三层价值锚定，实现"任何互助都能被量化、流转、兑付"。
   1.2 一句话描述
   用户可以像发朋友圈一样发布任务，像使用多币种钱包一样管理信用资产，用自己发行的"个人货币"或团体币奖励他人，构建独特的社交价值网络。
   1.3 核心价值主张
   
   对C端用户：让微小劳务（取快递、问问题、陪逛街）立即获得可量化的信用凭证，这些凭证可在社交圈内流通
   
   
   对小团体（家庭/社团）：快速建立内部经济系统，通过任务-奖励机制降低管理成本
   
   
   对平台：成为去中心化劳务的结算层，通过交易手续费和世界币充值盈利
   















增发：无上限，无成本，但显示"增发记录"供市场参考


转账：单向赠与或任务结算，不可主动"出售"（防证券化）

团体币 (Guild)：

铸造：仅团体管理员可操作，无上限


兑付：团体商店内兑换商品/服务，管理员线下履约


退出机制：成员退团时，团体币可选择：

o
按当前汇率兑换为世界币（需管理员预设储备）
o
o
转让给团内其他成员
o
o
作废（清零）
o

   }
   -- 货币持有表 (多对多关系)
   Holding {
   holder_id: FK(User),
   currency_type: Enum['personal', 'guild'],
   currency_key: String,        -- user_id 或 guild_id
   amount: Decimal,
   updated_at: Timestamp
   }
   -- 团体表
   Guild {
   guild_id: PK,
   name: String,
   admin_id: FK(User),
   type: Enum,
   treasury_shell: Decimal,     -- 世界币储备（可选）
   guild_token_issued: Decimal, -- 团体币总发行量
   created_at: Timestamp
   }
   -- 任务表 (统一)
   Task {
   task_id: PK,
   creator_id: FK(User),
   currency_type: Enum['shell', 'personal', 'guild'],
   currency_guild_id: FK(Guild)?, -- 仅guild类型
   reward_amount: Decimal,
   description: Text,
   location: GeoPoint,
   radius: Integer,             -- 可见范围米数
   status: Enum,
   executor_id: FK(User)?,
   escrow_id: String?,          -- 世界币托管订单号
   created_at: Timestamp,
   expired_at: Timestamp
   }
   -- 交易/转账记录 (统一账本)Transaction {
   tx_id: PK,
   task_id: FK(Task)?,          -- 关联任务(可选)
   from_user: FK(User),
   to_user: FK(User),
   currency_type: Enum,
   amount: Decimal,
   tx_type: Enum['task_reward', 'transfer', 'deposit', 'withdrawal'],
   status: Enum['pending', 'success', 'failed'],
   created_at: Timestamp
   }
   4.2 关键业务逻辑
   铸币逻辑 (个人币)：
   复制
   function mintPersonalToken(user_id, amount):
   user = User.find(user_id)
   user.my_token_issued += amount
   -- 同步创建holding记录，自己持有新增部分(可选)或直接转给他人
   任务结算逻辑：
   复制
   function settleTask(task_id):
   task = Task.find(task_id)
   if task.currency_type == 'shell':
   releaseEscrow(task.escrow_id, task.executor_id)
   else if task.currency_type == 'personal':
   transfer(task.creator_id, task.executor_id, 'personal', task.creator_id, task.reward_amount)
   else if task.currency_type == 'guild':
   transferGuildToken(task.currency_guild_id, task.executor_id, task.reward_amount)

7. 风控与合规
   7.1 金融监管合规
   
   世界币定性：预付式消费积分，受《单用途商业预付卡管理办法》约束
   
   o
   设置充值上限（单用户5000元）
   o
   o
   不可用于赌博、博彩
   o
   o
   提现收重税（20%）鼓励内部流通
   o
   
   个人币/团体币定性：债权凭证或虚拟商品，禁止法币交易对显示（不显示"1小明币=0.1元"）
   
   7.2 平台风险防控
   防欺诈：
   
   世界币任务：强制托管，防止发布者跑单
   
   
   个人币任务：限制单次赏金不超过发布者历史发行量的10%，防止骗劳务后注销
   
   
   冷启动保护：新用户24小时内不能发布世界币任务（防羊毛党）
   
   防通胀：
   
   团体币预警：当团体币M2增速超过世界币储备增速时，向管理员推送通胀警告
   
   
   个人币上限：建议设置个人币发行软上限（如10000枚），超限需解锁
   
   内容安全：
   
   AI预审任务描述（敏感词、色情、政治）
   
   
   图片OCR识别（防二维码导流、防暴力）
   
   
   LBS任务人工抽查（首月100%人工复核，后续抽样）
   
   7.3 争议仲裁流程
   复制
   争议提起 → 双方举证（48小时） →
   ├─ 世界币任务：平台AI+人工判定，输家扣除5%仲裁费
   ├─ 个人币任务：社交曝光（差评标记），无强制划转
   └─ 团体币任务：团体管理员二次判定，或成员投票（DAO模式）
